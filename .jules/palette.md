## 2024-03-13 - Added ARIA attributes to search filters and sorting
**Learning:** Screen readers need dynamic context for repeated "Clear all" buttons in facet filters, otherwise they all sound identical. Also, custom sort dropdowns must properly signal their expanded/collapsed state using `aria-expanded` to allow keyboard/screen reader users to understand the UI's state.
**Action:** Always use dynamic labels (e.g., ``aria-label={`Clear all ${label} filters`}``) when multiple identical actions exist on a page, and always tie `aria-expanded` to the state variable controlling a custom dropdown's visibility.
## 2025-02-05 - Global search keyboard navigation mapping linear navigation over grouped dictionaries
**Learning:** The `useSearch` hook returns search results as a grouped dictionary (`Record<string, SearchIndexResult>`). Implementing linear UI features, like arrow-key keyboard navigation in the global search modal, requires mapping the grouped indexes into a single continuous `globalIdx` structure, or explicitly computing a `flatHits` array out of the `results` object. Additionally, lifting the active tab state out of the nested results list avoids mismatching states between what the input navigation receives versus what the results render.
**Action:** When implementing arrow-key navigation across grouped or dictionary-based results, calculate a continuous flat array (`flatHits`) using `useMemo` in the common parent component. Drive all selected indexing and navigation off of that flat array, mapping it back to the respective keys natively inside the `.map` function rather than depending on local array index parameters.
