## 2024-05-14 - Extract Nested Component for Stability
**Learning:** Found that `ToolbarButton` was defined inside the `FeedbackItemPage` component body. Because React completely unmounts and remounts components defined within other components on every parent render, interactions that triggered heavy state updates (like zoom or hover) were degrading performance and causing unnecessary DOM churn for list items/frequently repeated elements.
**Action:** Extract simple functional components to the top-level module scope to preserve DOM state across parent renders, and wrap them in `React.memo` if they depend on stable props to prevent unnecessary updates.
